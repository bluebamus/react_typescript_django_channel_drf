[ 내용 개요 ]
사용자로 하여금 실시간 메시징, 고품질 음성 채팅 및 강력한 서버 관리 도구를 제공하는 채팅 서비스를 만듭니다.
이 서비스는 안전성, 확장성 및 다양한 플랫폼에서 사용 가능해야 합니다.
목표는 사용자들이 연결하고 공유하며 협업할 수 있는 즐겁고 매력적인 커뮤니티 공간을 제공하는 것입니다.
[ 채팅 서버 관리 API ]
사양 목표: 다양한 쿼리 매개변수에 따라 서버 목록을 검색하는 뷰세트를 정의합니다.
API는 서버 목록을 검색하여 사용자에게 반환하고, 매개변수가 유효하지 않거나 사용자가 인증되지 않은 경우 적절한 오류를 발생시킵니다.
서버 목록은 ServerSerializer를 사용하여 직렬화되며, 요청 시 '회원 수'를 포함하게 됩니다.
[ ServerListViewSet의 개요 ]
서버 목록을 검색하는 역할을 합니다.
지원하는 쿼리 매개변수:
qty (정수)
category (문자열)
by_user (부울)
with_num_members (부울)
all_channels (부울)
by_serverid (정수)
'by_user' 매개변수가 참이고 사용자가 인증되지 않는 경우, AuthenticationFailed 오류를 발생시킵니다.
'category' 매개변수가 지정된 경우, 쿼리셋은 해당 카테고리의 서버만 포함해야 합니다.
'by_user'가 참일 때는 현재 사용자가 회원인 서버만 포함해야 합니다.
'qty' 매개변수가 지정된 경우, 쿼리셋은 지정된 수의 서버로 제한해야 합니다.
'with_num_members'가 참일 경우, 각 서버의 회원 수를 쿼리셋에 주석으로 추가해야 합니다.
'all_channels'가 참일 때 특정 'server_id'가 지정되면, 쿼리셋은 해당 서버의 채널만 포함해야 합니다.
'by_serverid' 매개변수가 지정된 경우, 쿼리셋은 해당 ID를 가진 서버만 포함해야 하고, 유효하지 않거나 존재하지 않을 경우 ValidationError를 발생시킵니다.
서버 목록은 ServerSerializer를 사용해서 직렬화되며, 'with_num_members'가 참일 경우 'num_members' 필드가 포함됩니다.
